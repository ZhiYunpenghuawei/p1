# RecIF SID层次化生成流程说明

## 1. SID层次结构

RecIF中的SID（Semantic ID）采用三级层次结构：

```
SID = [SC, SB, SA]
```

其中：

- SA：最低层语义ID（细粒度）
- SB：中间层语义ID
- SC：最高层语义ID（粗粒度）

---

# 2. SID二进制切分示例

假设一个SID由39 bit组成，每个层级占13 bit：

```
SID:

0000 0000 0000 11 | 0000 0000 0000 10 | 0000 0000 0000 01

        SC                   SB                   SA
       高位                 中间                 低位
```

因此：

```
SC = 0000000000011 = 3

SB = 0000000000010 = 2

SA = 0000000000001 = 1
```

每个层级占13 bit，因此取值范围：

```
2^13 = 8192
```

即：

```
SA ∈ [0,8191]

SB ∈ [0,8191]

SC ∈ [0,8191]
```

三个层级本质上都是独立的8192分类空间。

---

# 3. Prefix序列构造与层次偏置

由于SA、SB、SC处于不同语义层级，如果直接输入：

```
SA=1
SB=2
SC=3
```

模型无法区分三个数字分别代表哪一层。

因此RecIF引入层次偏置（Hierarchical Bias），将三个层级映射到不同区域：

```
SA offset = 0

SB offset = 8192

SC offset = 8192 × 2
```

因此：

## SA

保持原值




---

## SB

增加偏置：

```
SB'=SB+8192
```

例如：

```
SB=2

SB'=2+8192

=8194
```

---

## SC

增加偏置：

```
SC'=SC+8192×2
```

例如：

```
SC=3

SC'=3+16384

=16387
```

---

最终输入空间：

```
SA:

0 ~ 8191


SB:

8192 ~ 16383


SC:

16384 ~ 24575
```

因此三个层级被扩展到了统一的：

```
8192 × 3 = 24576
```

维度空间。

---

# 4. Prefix序列构造

原始SID：

```
[SC, SB, SA]
```

经过层次偏置后：

```
[
ctx,

SA,

SB+8192,

SC+8192×2,

...
]
```

例如：

原始：

```
SA=1

SB=2

SC=3
```

构造：

```
[
ctx,

1,

8194,

16387
]
```

这里：

- SA占用0~8191区域
- SB占用8192~16383区域
- SC占用16384~24575区域

因此模型能够明确感知不同层级的信息。

---

# 5. Backbone与Head预测过程

Prefix序列输入Transformer Backbone：

```
Prefix Tokens

      |
      ↓

Transformer Backbone

      |
      ↓

Hidden Representation
```

随后进入三个独立Head：

```
                 Backbone

                    |
       ----------------------------
       |             |             |
     SA Head       SB Head       SC Head
```

每个Head负责预测一个层级：

```
SA Head:

hidden_dim → 8192


SB Head:

hidden_dim → 8192


SC Head:

hidden_dim → 8192
```

因此Head输出重新回到了单层语义空间：

```
SA prediction:

0~8191


SB prediction:

0~8191


SC prediction:

0~8191
```

---

# 6. Head输出后的重新拼接

注意：

Head输出时已经去除了层次偏置。

例如：

模型预测：

```
SB = 8
```

Head输出：

```
8
```

但是如果重新拼接到Prefix中，需要恢复层级空间：

```
SB_input = 8 + 8192
```

得到：

```
8200
```

同理：

预测：

```
SC=6
```

Head输出：

```
6
```

拼接时：

```
SC_input = 6 + 8192×2
```

得到：

```
16390
```

因此：

```
Head输出:

SA

SB

SC


重新进入Prefix:

SA

SB+8192

SC+8192×2
```

---

# 7. 层次化生成关系

RecIF并不是独立预测SA、SB、SC，而是利用层次结构递归生成。


## 多个SA生成SB

例如：

```
SB = 2
```

对应多个SA：

```
SB=2

 |
 +---- SA=1
 |
 +---- SA=5
 |
 +---- SA=8
 |
 +---- SA=10
```

即：

```
多个SA

↓

一个SB
```

---

## 多个SB生成SC

例如：

```
SC=3
```

包含：

```
SC=3

 |
 +---- SB=2
 |
 +---- SB=4
 |
 +---- SB=7
```

每个SB下面又包含多个SA。

因此：

```
多个SA

↓

SB

↓

多个SB

↓

SC
```

形成完整的层次化SID生成过程。

---

# 8. 总体流程总结

```
原始SID

[SC, SB, SA]

        |
        ↓

二进制切分

SC=3
SB=2
SA=1

        |
        ↓

加入层次偏置

SA

SB+8192

SC+8192×2

        |
        ↓

构造Prefix Sequence

[ctx, SA, SB+8192, SC+16384]

        |
        ↓

Transformer Backbone

        |
        ↓

三个Head预测

SA:8192

SB:8192

SC:8192

        |
        ↓

重新加入层次偏置

SA

SB+8192

SC+16384

        |
        ↓

生成新的层次化SID
```

RecIF通过层次偏置将三个独立的8192空间扩展为统一的24576空间，使模型能够同时学习：

- SA内部语义关系
- SB与多个SA之间的聚合关系
- SC与多个SB之间的层次关系

最终实现从细粒度SA到高层语义SC的层次化生成。
